Security

------------------------------------------------------------
These are key security rules so websites dont get hacked by people wanting data or other things
----
HTTPS & SSL/TLS: Use SSL/TLS encryption for all data in transit to prevent interception.
----
Strong Authentication: Implement Multi-Factor Authentication (MFA) for admin access and strong password hashing for users.
----
Software Updates & Patching: Regularly update the web server, CMS, plugins, and dependencies to patch known vulnerabilities.
----
Web Application Firewall: Deploy a Web Application Firewall to block common attacks like SQL injection.
----
Secure Coding Practices: Sanitize all user input, validate data on both client and server sides, and use parameterized queries.
----
Data Encryption at Rest: Encrypt sensitive information in the database using strong encryption algorithms.
----
Regular Audits & Pentesting: Conduct vulnerability scans and penetration testing.
----
Secure Hosting: Use a reputable hosting provider that offers built-in security, such as DDoS protection and malware scanning
----
